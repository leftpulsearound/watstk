# LinkVault Resource Aggregator

LinkVault 是一个面向开发者、技术研究人员与内容策展人的轻量级外链资源汇总与导航系统。该项目定位于将分散在多个子域名下的技术文章、教程、案例分析及开发日志进行集中索引与分类展示，帮助用户快速发现、检索和访问高价值的外部技术内容。

项目本身不存储任何原始文章内容，仅提供结构化的元数据索引与友好的浏览界面。目标用户包括需要持续追踪技术动态的软件工程师、撰写技术博客的编辑、以及进行竞品分析或市场调研的产品经理。通过 LinkVault，用户可以将原本零散的书签、收藏夹或内部 Wiki 中的外链转化为可检索、可分类、可共享的组织化知识资产。

## 功能概览

**多源异构数据接入**：支持从不同子域名（如移动端子域名、H5 子域名）自动抓取文章元数据，并统一转换为标准索引格式。

**分类与标签体系**：基于文章 ID 与来源域名自动生成分类维度，支持自定义标签，便于按技术领域、难度等级或发表日期进行筛选。

**全文检索与过滤**：内置轻量级倒排索引，支持对文章标题、摘要及来源进行关键词搜索，并可按来源域名、ID 范围进行过滤。

**响应式浏览界面**：提供桌面端与移动端适配的列表视图与详情预览面板，无需跳转原始链接即可阅读摘要信息。

**定期索引更新机制**：支持通过定时任务或 Webhook 触发增量更新，确保索引库与源站内容保持同步。

**访问统计与热度排序**：记录每个外链的点击次数与最近访问时间，支持按热度、更新时间或相关性排序。

**开放 API 接口**：提供 RESTful API 供第三方工具查询索引数据，便于集成至内部仪表盘或自动化工作流。

**数据导出与备份**：支持将索引数据导出为 JSON、CSV 或 Markdown 格式，便于离线分析或迁移。

## 应用场景

**技术团队内部知识库建设**：技术团队可将 LinkVault 部署为内部知识导航页，统一收录团队推荐的博客文章、官方文档与社区讨论链接，新成员入职时可快速了解团队关注的技术资源分布。

**个人开发者的阅读清单管理**：开发者可将日常浏览中发现的优质文章通过浏览器插件一键提交至 LinkVault，系统自动去重并归类，替代零散的浏览器书签，形成可持续维护的阅读清单。

**技术社区的内容聚合展示**：技术社区或开源项目维护者可使用 LinkVault 生成项目周边的外链广场，展示相关教程、案例研究及生态工具，降低社区用户的信息获取成本。

**竞品动态监控与情报收集**：市场分析人员可配置针对特定子域名或文章 ID 范围的监控规则，当新增高价值内容时，LinkVault 通过邮件或 IM 机器人推送通知，辅助竞品动态跟踪。

## 快速开始

以下步骤帮助您在本地环境中快速启动 LinkVault 服务。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkvault.git

# 进入项目目录
cd linkvault

# 安装依赖（使用 pip 和虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化数据库并导入示例索引数据
python manage.py init_db
python manage.py load_fixtures

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8080
```

访问 `http://localhost:8080` 即可进入 LinkVault 仪表板。首次启动将自动创建默认管理员账号，请通过控制台输出的初始化密码进行登录。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Python | 3.9 及以上 | 核心运行时环境，建议使用 3.11 LTS 版本 |
| SQLite | 3.35 及以上 | 默认嵌入式数据库，用于存储索引元数据，生产环境可切换至 PostgreSQL |
| Redis | 6.2 及以上 | 用于缓存热点查询结果与暂存异步任务队列（可选，默认使用内存缓存） |
| Node.js | 18.x LTS | 仅用于前端资源构建（Sass 编译与 JavaScript 打包），生产环境可预编译 |
| Nginx | 1.20 及以上 | 生产环境推荐反向代理服务器，用于静态资源加速与负载均衡 |
| Cron / systemd-timer | 任意版本 | 用于调度周期性索引更新任务（如每日凌晨全量检查） |
| Git | 2.25 及以上 | 用于版本管理与通过钩子触发增量索引更新 |
| OpenSSL | 1.1.1 及以上 | 用于生成 API 访问令牌的签名与验证 |
| docker / docker-compose | 20.10 及以上 | 若使用容器化部署方式，需提供 Docker 环境（生产推荐） |

## 常见问题

**Q：索引更新时遇到目标站点返回 429 或 503 状态码，系统如何处理？**

A：适配器层内置指数退避重试策略，初始等待 1 秒，最大重试 3 次。若仍失败，该 URL 将被记录至失败队列，并在下一次全量更新时重新尝试。管理员可通过管理后台查看失败列表并手动触发重试。

**Q：能否自定义数据源，添加非 blog.occzl.cn 域名的链接？**

A：可以。您需要实现 `internal/adapter/parser.go` 中定义的 `Parser` 接口，并在 `fetcher.go` 中注册新的域名匹配规则。我们欢迎贡献通用型解析器，以支持更多主流技术博客平台。

**Q：生产环境使用 PostgreSQL 替代 SQLite 时，需要注意哪些迁移事项？**

A：LinkVault 提供自动迁移脚本，启动时若检测到 PostgreSQL DSN 环境变量，会自动创建表结构并迁移现有数据。建议在维护窗口执行迁移，并提前使用 `scripts/backup.sh` 备份 SQLite 文件。迁移完成后，请更新 `config.prod.yaml` 中的数据库连接池参数，避免连接数耗尽。
