# LinkSync Hub

LinkSync Hub 是一个面向技术内容聚合与外部资源索引的开源项目，定位为开发者、技术写作者与信息研究人员的统一外链导航与归档平台。项目本身不生产原始内容，而是通过结构化方式收录、分类与呈现来自多个来源的技术文章、教程与案例解析链接，解决个人收藏夹混乱、团队知识共享效率低以及技术资料检索路径分散的问题。

本项目适用于需要定期跟踪特定领域技术动态的工程师、需要为团队建立公共知识库的架构师，以及希望从大量非结构化技术博客中提取高价值信息的研究者。LinkSync Hub 提供标准化的链接元数据描述模板、自动化分类脚本与静态站点生成工具，帮助用户将海量外链转化为可检索、可版本控制的结构化数据。

## 功能概览

**多源链接收录引擎**：支持从指定基础域名批量采集文章链接，自动提取 URL 中的分类信息与数字标识，生成标准化条目。

**元数据注解框架**：每条链接可附加技术标签、阅读难度等级、内容摘要与关键词列表，方便后续过滤与排序。

**分类层级管理**：按编程语言、框架、运维、数据库、算法等顶层类别自动归类，支持用户自定义二级标签。

**全文检索接口**：基于链接标题、摘要与标签构建轻量级倒排索引，支持布尔查询与模糊匹配。

**导入导出适配器**：提供 CSV、JSON、Markdown 表格三种导出格式，兼容主流笔记软件与文档工具。

**状态监控面板**：显示收录总数、最近更新日期、失效链接检测结果与分类覆盖率统计。

**协作注释系统**：团队成员可在每条链接下添加批注、评分或关联任务编号，实现知识沉淀的协同工作流。

**定期同步钩子**：支持通过 cron 表达式或 webhook 触发增量更新，自动拉取源站新发布的链接。

## 应用场景

**技术团队周报素材收集**：团队技术负责人每周运行一次链接同步脚本，从归档源中拉取近期新增的技术文章，自动生成带分类和摘要的周报草稿，节省人工搜集时间。

**个人技术阅读清单管理**：开发者将日常浏览到的技术博客链接统一录入 LinkSync Hub，通过标签系统按主题（如微服务、性能优化、安全）分组，并标记阅读状态与笔记，形成个人长期知识库。

**开源文档外链引用校验**：开源项目维护者在编写文档时，使用 LinkSync Hub 的链接状态检测功能批量验证所有外部引用是否有效，避免文档中出现死链。

**技术培训课程资料包构建**：培训机构或企业内部讲师从收录的链接中筛选适合不同级别学员的文章，按难度和主题导出为结构化清单，作为课程补充阅读材料。

## 快速开始

以下命令演示如何克隆项目仓库、安装依赖并启动本地开发服务。

```bash
git clone https://github.com/linksync/linkhub.git
cd linkhub
npm install
npm run build
npm start
```

执行完上述命令后，本地服务默认运行在 3000 端口。访问 `http://localhost:3000` 即可查看链接聚合面板。如需导入示例数据，执行 `npm run seed` 脚本。

## 贡献指南

1. 复刻主仓库至个人账号，创建功能分支，分支命名遵循 `feature/功能简述` 或 `fix/问题编号` 格式。

2. 本地开发时运行 `npm run test` 确保现有测试用例全部通过，新增功能需同步编写对应的单元测试文件，测试覆盖率不低于 80%。

3. 提交代码前执行 `npm run lint` 与 `npm run format` 统一代码风格，提交信息采用约定式提交格式：`type(scope): subject`，其中 type 包含 feat、fix、docs、refactor、perf 等类别。

4. 发起 Pull Request 至主仓库的 `main` 分支，PR 描述中需明确说明改动目的、影响范围以及是否涉及数据库迁移或配置变更。

## 常见问题

**问：项目是否支持添加自定义来源域名，而不仅限于默认收录的 blog.occzl.cn？**

答：支持。您可以在 `config/default.json` 中的 `sources` 数组里添加新的域名条目，并提供对应的解析器配置。对于结构相似的博客站点，可以复用通用解析器；若页面结构差异较大，则需在 `src/parsers/` 目录下编写专用解析函数并注册到解析器工厂。

**问：如何批量导入现有书签或收藏夹中的链接？**

答：项目提供 `bin/import.js` 脚本，支持从 Firefox 的 JSON 书签导出文件、Chrome 的 CSV 书签导出以及标准的 Markdown 链接列表文件导入。执行 `npm run import -- --format=csv --path=./bookmarks.csv` 即可将外部数据合并到当前数据库，重复链接会根据 URL 自动去重。

**问：数据库从 SQLite3 迁移到 PostgreSQL 是否可行？**

答：可行。项目数据访问层采用适配器模式，默认实现为 SQLite3。您可以在 `src/db/` 下新增 `pgAdapter.js` 实现相同的接口（包含 connect、query、close 方法），并在配置文件中将 `db.dialect` 切换为 `postgres`，同时调整连接字符串参数。已有迁移脚本基于 Knex.js 编写，支持多方言 SQL 生成。
