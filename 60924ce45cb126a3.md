# Hyperlink Indexer

Hyperlink Indexer 是一个面向技术内容聚合与结构化外链管理的开源工具集。该项目定位于为开发者、技术博主、文档维护者以及知识库管理员提供一套轻量级的链接收集、分类、校验与展示方案，帮助用户从分散的原始链接数据中快速生成可维护的 Markdown 索引文档。

本项目不依赖复杂的数据库或前端框架，核心围绕静态 Markdown 生成与纯文本处理展开，适合作为 CI/CD 流水线中的文档生成环节，或作为个人知识管理（PKM）工作流中的链接清洗与归档工具。Hyperlink Indexer 默认以批次（Batch）为单位组织链接资源，每个批次可包含数百个外部链接，并自动生成符合项目规范的 README 导航页。

## 功能概览

- **批量链接收录**：支持一次性导入数十至数百个原始 URL，自动去重并按来源域名归类。

- **结构化文档生成**：根据预定义章节模板，将链接列表嵌入项目 README，生成符合开源社区阅读习惯的 Markdown 文件。

- **协议与格式校验**：自动检测 URL 协议头（http / https）和域名格式，对缺失协议或包含非法字符的条目输出警告日志。

- **分类标签注入**：允许用户为每个链接或每个批次添加自定义标签（如 `backend`、`devops`、`frontend`），并在生成的索引中按标签过滤展示。

- **ASCII 目录树输出**：项目本身提供完整的文件结构树注释，帮助新贡献者在三分钟内理解代码布局。

- **纯净 Markdown 输出**：所有生成内容均以纯 Markdown 格式输出，不嵌入 HTML 标签或 JavaScript，确保在 GitHub、GitLab、Gitee 等主流平台上的渲染一致性。

- **多批次管理**：通过批次编号（如第 22/56 批）区分不同来源或不同时期的链接集合，便于历史回溯与增量更新。

## 应用场景

**技术博客外链归档**：技术博主在日常写作中引用大量外部参考链接，使用 Hyperlink Indexer 可以将这些链接按文章批次整理成独立的 README 文档，方便读者查阅原文出处，同时降低博客正文的冗余超链接数量。

**开源项目文档中心导航**：大型开源项目通常拥有数十个相关子项目、工具链、示例仓库和在线演示地址。维护者可将这些链接导入 Hyperlink Indexer，生成统一的资源导航页，放置于项目根目录或 `docs` 文件夹下，供社区成员快速定位。

**企业知识库链接治理**：企业内部 Wiki 常常积累大量失效或重复的外部链接。运维团队可定期将待审核的链接列表通过本工具生成索引快照，结合人工审查后发布为官方推荐资源清单，提升知识库的权威性与可读性。

**离线文档打包预处理**：在生成离线文档包（如 Dash、Zeal 或 AirPocket 格式）之前，使用 Hyperlink Indexer 将分散在多个 Markdown 文件中的外部链接统一提取并归一化为单一索引文件，方便后续链接有效性批量检查。

## 快速开始

以下指令适用于 Linux / macOS / Windows（WSL 或 Git Bash）环境。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/hyperlink-indexer.git

# 进入项目目录
cd hyperlink-indexer

# 安装依赖（Python 3.8+，使用 pip 安装）
pip install -r requirements.txt

# 运行索引生成器，指定批次编号和原始链接文件
python indexer.py --batch 22 --input raw_urls_22.txt --output README.md
```

执行上述命令后，程序将读取 `raw_urls_22.txt` 中的链接列表（每行一个 URL），按照内置模板生成完整的 README.md 文件，并自动填充「资源列表」章节。

## 项目结构

```
hyperlink-indexer/
├── indexer.py                # 主入口脚本，解析命令行参数并调度生成流程
├── config.yaml               # 全局配置文件，定义输出目录、默认批次号、章节开关
├── requirements.txt          # Python 依赖声明文件
├── README.md                 # 当前文档（由工具自身生成或手动维护）
├── CONTRIBUTING.md           # 贡献者行为准则与提交指南
├── LICENSE                   # MIT 许可证全文
├── core/                     # 核心逻辑模块
│   ├── parser.py             # URL 解析器，负责去重、格式归一化与非法字符过滤
│   ├── generator.py          # Markdown 生成器，基于模板引擎渲染各章节
│   └── validator.py          # 链接存活检测模块（可选，依赖 requests）
├── templates/                # 文档模板目录
│   ├── base.md               # 基础 Markdown 骨架，包含固定章节占位符
│   └── batch_section.md      # 批次资源列表的子模板，用于循环输出链接条目
├── tests/                    # 单元测试目录
│   ├── test_parser.py        # 解析器单元测试用例
│   ├── test_generator.py     # 生成器输出正确性测试
│   └── fixtures/             # 测试用静态数据（样本链接列表）
│       └── sample_urls.txt
├── docs/                     # 用户文档
│   ├── quickstart.md         # 快速入门教程
│   ├── configuration.md      # 配置项详解
│   └── batch-rules.md        # 批次编号规范与链接排序算法说明
└── scripts/                  # 辅助运维脚本
    ├── batch_import.sh       # 从外部 CSV 批量导入链接的 Shell 包装器
    └── check_links.sh        # 周期性链接有效性检查的 Cron 脚本
```

## 常见问题

**问：工具是否支持 HTTPS 链接与 HTTP 链接混合输入？**  
答：支持。`parser.py` 模块会保留每个 URL 的原始协议头，不会强制转换 http 为 https 或反之。但工具会输出警告信息提示用户注意混合协议可能带来的内容安全问题，尤其是在内网环境中。

**问：生成的 README.md 是否可以手动编辑后再提交？**  
答：可以。工具生成的 Markdown 文件是纯文本格式，用户完全可以在生成后手动调整章节顺序、增删内容或修改措辞。但若需要重新运行生成器覆盖文件，建议先备份手动修改的部分，因为默认模式下生成器会整体替换目标文件。若希望保留特定章节，可在 `config.yaml` 中设置 `preserve_manual_sections: true`。

**问：链接数量很大（例如超过 1000 条）时，生成速度如何？**  
答：在常规硬件（8GB 内存、四核 CPU）上，处理 1000 条链接的解析与生成耗时通常在 2 秒以内，主要瓶颈在于文件 I/O 和可选链接存活检测的网络请求。若启用 `--check` 参数进行实时存活探测，耗时将取决于网络延迟和链接数量，建议在后台异步执行或使用 `--check-timeout` 参数调低超时阈值。
