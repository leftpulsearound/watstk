# ResourceHub

ResourceHub 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源导航与外链聚合平台。该项目旨在解决技术信息碎片化、优质内容难以追溯的问题，通过结构化目录与人工筛选机制，将散落于网络各处的技术文章、教程、案例及工具文档进行集中归类与索引。

ResourceHub 并非一个传统的爬虫站或自动化采集系统，而是一个强调可维护性与可读性的静态资源索引库。它适用于个人知识管理、团队技术沉淀、开源项目文档外链补充以及技术博客的延伸阅读列表等场景。项目本身不存储任何第三方内容，仅提供指向原始来源的稳定链接，确保内容的版权归属与实时性。

## 功能概览

**按技术领域分类索引** 每个收录链接均被打上框架、语言、数据库或工程实践等标签，支持快速按领域筛选。

**文章质量预评级** 基于来源域名、作者历史内容及阅读量等维度，对入库链接进行初步的参考价值标注。

**多级目录树导航** 仿照技术文档站点结构，提供从入门到进阶的层级化浏览路径，降低信息过载。

**全文标题与摘要搜索** 支持对已收录文章的标题与手动编写的摘要进行关键词检索，不依赖外部搜索引擎。

**每日自动链接健康检查** 通过定时任务检测已收录链接的可访问性，自动标记失效链接，保证资源列表的有效率。

**用户提交与反馈机制** 允许社区成员通过 Issue 或 PR 提交新的优质链接，经审核后纳入主库。

**导入导出标准格式** 支持将资源列表导出为 JSON、CSV 或 Markdown 表格，便于迁移至其他知识管理工具。

## 应用场景

**技术团队内部知识库构建** 团队 Leader 或技术文档负责人可将 ResourceHub 作为团队每日阅读清单的基础，统一团队成员的技术视野与学习路径，减少重复寻找资料的时间成本。

**开源项目补充文档外链** 当开源项目的 README 或官方文档需要引用大量外部参考资料时，可将 ResourceHub 中对应的分类链接作为附录或脚注，避免主文档过于冗长。

**技术博客与 newsletter 素材池** 技术博主或周刊编辑可定期从 ResourceHub 的更新列表中获取近期热门或深度技术文章，快速整理出高质量的内容推荐合集。

**个人技术成长路径规划** 初学者可依照 ResourceHub 的层级目录，从基础概念开始逐步深入特定技术栈，形成系统化的自学习惯，而非零散地浏览碎片信息。

## 快速开始

以下步骤将帮助您在本地环境中快速启动 ResourceHub 实例。

```bash
# 克隆代码仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 安装项目依赖（基于 Node.js 22 LTS 与 pnpm）
pnpm install

# 构建静态资源索引并启动开发服务器
pnpm run build
pnpm run start
```

启动成功后，访问控制台输出的本地地址（默认 http://localhost:3000）即可浏览资源列表。

```
│   ├── ui/                       # 前端静态资源
│   │   ├── templates/            # 服务端渲染模板（EJS）
│   │   ├── static/               # CSS、JavaScript、图片等静态资产
│   │   └── components/           # 可复用的前端 UI 组件（卡片、列表、分页）
│   └── utils/                    # 通用工具函数
│       ├── logger.js             # 结构化日志输出（支持 JSON 与 plain）
│       └── db-client.js          # SQLite 数据库连接池与查询封装
├── tests/                        # 单元测试与集成测试用例
│   ├── unit/                     # 针对核心函数的隔离测试
│   └── integration/              # 端到端 API 测试与健康检查模拟
├── docs/                         # 完整项目文档（含用户手册与 API 参考）
├── scripts/                      # 运维与辅助脚本
│   ├── import-csv.js             # 从 CSV 批量导入历史链接
│   └── export-json.js            # 将当前索引导出为标准 JSON 格式
├── .env.example                  # 环境变量模板（端口、数据库路径、日志级别）
├── package.json                  # Node.js 项目清单与依赖声明
└── README.md                     # 项目入口文档（即本文档）
```

## 贡献指南

我们欢迎社区贡献者以多种形式参与 ResourceHub 的建设，包括但不限于提交新资源链接、修正现有链接错误、改进文档以及完善分类体系。所有贡献均需遵循以下步骤。

1.  在 GitHub 上 Fork 本仓库至个人账户，并克隆到本地开发环境。
2.  创建新的功能分支，分支名称需简明描述变更内容，例如 `feat/add-react-hooks-links` 或 `fix/broken-urls-202607`。
3.  按照 `docs/maintainer/contribution-checklist.md` 中的条目逐项核对所提交链接的质量，确保链接有效、内容与分类匹配且不重复。
4.  提交变更时使用语义化提交信息格式，例如 `feat: 新增 10 条 TypeScript 进阶文章链接` 或 `docs: 更新部署指南中的 Nginx 配置示例`。
5.  创建 Pull Request 并填写标准模板，描述变更动机、影响范围以及测试情况，等待维护者审阅。

## 常见问题

**Q：ResourceHub 是否存储第三方文章的内容副本？**
A：不存储。ResourceHub 仅保存链接、标题、摘要与分类元数据，所有内容著作权归属原作者，访问时直接跳转至原始来源。我们尊重版权，不进行任何形式的全文转载或镜像。

**Q：链接健康检查的周期是多久？失效链接如何处理？**
A：系统默认每 72 小时执行一次全局检查。连续两次检查失败的链接会被标记为“暂不可用”，并从默认浏览视图中隐藏，但保留在数据库中。维护者会每季度人工复核一次失效链接列表，尝试寻找可替代的镜像或更新后的地址。

**Q：普通用户如何提交新链接？是否需要注册账号？**
A：用户无需注册即可通过项目首页的“提交推荐”表单输入链接与推荐理由。提交后系统会生成一个工单号，用户可通过该工单号在“提交状态”页面查询审核进度。审核通过后，链接会在下一个构建周期纳入索引。
