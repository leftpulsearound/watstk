# TechLink Navigator

TechLink Navigator 是一个面向技术研究者、开发者与内容策展人的开源外链资源聚合平台。该项目不生产内容，而是以高度结构化的方式收录来自特定技术博客站点的深度文章链接，并提供分类导航、快速检索与元数据标注能力。项目定位为“技术文章的入口索引系统”，帮助用户在大量分散的博客文章中快速定位到与后端架构、数据库调优、DevOps 实践、前端工程化等主题相关的高质量讨论。

项目采用静态站点生成逻辑，所有链接资源以纯文本数据文件存储，支持通过脚本进行自动化校验、去重与可用性检查。核心使用场景包括个人知识库的素材收集、技术团队周报的参考资料汇编、以及开源文档的外部引用来源管理。当前批次为第 26/56 批资源整合，共计收录 180 条来自 h5.blog.occzl.cn 的深度技术文章链接，覆盖性能分析、设计模式、容器化、监控告警等十余个技术子领域。

## 功能概览

**结构化链接索引**：按文章 ID 对原始 URL 进行规范化存储，每条记录保留原始路径与文件扩展名，确保与源站数据完全一致。

**批量导入与校验**：提供 Python 和 Shell 脚本，支持批量导入 URL 列表，自动校验 HTTP 状态码，标记失效链接并生成报告。

**分类标签生成**：基于文章 ID 段与路径特征，自动生成技术领域标签（如 database、network、security），便于后续过滤。

**全文元数据模拟**：为每条链接生成占位元数据（发布时间、预估阅读时长、关键词），满足文档检索系统的数据模型要求。

**静态站点预览**：内置轻量级 HTTP 服务器，可在本地启动预览页面，以卡片网格形式展示所有链接及其标签。

**导出与集成**：支持将链接列表导出为 JSON、CSV 或 Markdown 表格格式，便于嵌入其他文档系统或 Wiki 平台。

**定时健康检查**：通过 cron 任务定期检查所有外链的可访问性，自动更新状态标记，保障资源列表的长期有效性。

## 应用场景

技术团队内部知识库建设：团队技术负责人可定期将本项目的链接列表导入内部 Confluence 或 Notion 页面，作为每周技术荐读的素材池，减少重复搜索成本。

个人技术博主参考文献管理：独立博主在撰写技术深度文章时，可通过本项目的分类标签快速找到与主题相关的外部引用来源，提升文章的可信度与信息密度。

开源项目文档外部链接治理：开源项目维护者可将本项目作为文档中的“扩展阅读”章节的数据源，通过脚本自动生成外部链接附录，并利用健康检查功能避免文档中出现死链。

技术社区内容策展：社区运营人员可基于本项目的链接集合，策划每月技术文章精选，按标签分组后发布为 newsletter 或专题推荐页。

## 快速开始

以下命令演示了从克隆仓库到启动本地预览服务的完整流程。

```bash
# 克隆仓库
git clone https://github.com/techlink-navigator/tln-core.git
cd tln-core

# 安装依赖（Python 3.9+ 环境）
pip install -r requirements.txt

# 运行本地预览服务器（默认监听 8080 端口）
python scripts/serve.py --port 8080
```

执行上述步骤后，打开浏览器访问 `http://localhost:8080` 即可看到当前批次全部 180 条链接的卡片式列表页面。如需重新生成索引数据，可运行 `python scripts/build_index.py --input data/urls_26.txt --output dist/`。

## 项目结构

## 常见问题

**Q：为什么所有链接都来自同一个域名？是否可以添加其他来源？**

A：当前项目按批次组织资源，第 26 批次专门收录 h5.blog.occzl.cn 下的文章，目的是保证数据源的一致性和可追溯性。项目本身支持多数据源，您可以在 `data/` 目录下创建新的批次文件（如 `urls_27_other_source.txt`），并修改 `scripts/build_index.py` 中的源读取逻辑来合并多个来源。未来版本将提供配置文件以声明多源输入。

**Q：链接健康检查报告显示部分链接返回 404，应该如何处理？**

A：健康检查脚本 `check_health.py` 会生成 `health_report.log` 文件，其中记录了所有非 200 状态码的链接。对于 404 链接，建议您首先访问原 URL 确认内容是否已迁移或删除。若内容已迁移，请更新 `data/` 中的对应 URL 为新的有效地址；若内容已永久删除，可从批次文件中移除该行，并在提交 PR 时备注说明。项目维护者也会定期根据健康报告清理失效资源。
