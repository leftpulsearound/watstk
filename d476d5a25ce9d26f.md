# WebArchive Link Hub

WebArchive Link Hub 是一个面向技术研究者、数据归档工作者和内容分析师的轻量级外链汇总与导航系统。该项目不生产内容，也不存储任何原始数据，而是以结构化的方式对分散在网络各处的技术文章、案例分析、开发文档和历史快照进行归类与索引，帮助用户快速定位特定主题下的高质量外链资源。

项目定位为纯静态链接聚合层，适用于个人知识管理、团队技术雷达构建、以及公开情报线索整理等场景。通过统一的条目格式和分类标签，用户可以在海量链接中按主题、时间或来源进行筛选与检索，显著降低信息碎片化带来的认知负载。本仓库包含第 33/56 批共计 180 个外链条目，所有链接均保留原始来源地址与访问协议，确保可追溯性与数据完整性。

## 功能概览

**按主题分类索引** 每批次链接依据文章标题、URL 路径特征和来源域名自动归入预设分类，支持技术栈、业务领域、运维实践等维度。

**原始地址透传** 所有外链保留原始 URL 中的大小写、文件扩展名及查询参数，确保跳转后页面内容与作者发布时完全一致。

**批次化版本管理** 采用批次编号（33/56）组织资源，每个版本对应一次爬取或人工提交周期，便于跟踪链接增删与状态变更。

**轻量级元数据标注** 每条记录附带文章 ID、来源域名及文件类型（如 .sHtML），为后续自动化分析提供结构化字段。

**纯静态部署支持** 项目生成 HTML 目录页与 JSON 索引文件，可托管于任何 Web 服务器或 CDN，无需数据库或后端运行时。

**可扩展分类模板** 提供 YAML 格式的分类映射配置文件，用户可根据需要新增或调整分类规则，重新生成导航页面。

**链接状态检测接口** 预留外部健康检查工具的对接规范，可定期扫描链接可用性并在导航页标记异常条目。

## 应用场景

**技术文献回溯** 研究人员可通过本索引快速定位 wap.blog.occzl.cn 上特定 ID 的历史文章，例如查找编号 1366969 或 84794 对应的技术笔记，无需手动翻页或依赖站内搜索。

**外部资源审计** 企业合规团队可利用本批次链接清单，对外链指向的第三方内容进行定期复核，确保引用资源未发生篡改或迁移，满足内部审计要求。

**开发文档聚合** 开发团队可将本仓库作为技术选型参考库，按文章编号排序浏览最新收录的案例分享，例如编号 29772 和 34159 可能涉及架构设计或性能调优话题。

## 快速开始

以下命令将在本地克隆仓库、安装依赖并启动静态导航服务。

```bash
git clone https://github.com/archive-labs/webarchive-link-hub.git
cd webarchive-link-hub
# 安装依赖（Node.js 16+ 环境）
npm install
# 生成静态站点（输出至 dist/ 目录）
npm run build
# 启动本地预览服务（默认端口 8080）
npm run serve
```

## 贡献指南

1. 新增外链批次时，在 data/ 目录下创建 raw-links-{批次号}.txt 文件，每行写入一个完整 URL（保持原始协议与大小写），并更新 config/batch-{批次号}.yaml 中的元数据字段。

2. 修改 config/categories.yaml 以调整分类关键词映射规则，语法遵循 YAML 1.2 规范。修改后运行 npm run validate 检查配置文件格式正确性。

3. 提交变更前执行 npm test 确保所有单元测试通过，特别是 URL 格式校验与分类覆盖度测试。测试报告输出至 test/reports/ 目录。

4. 发起的 Pull Request 需包含变更说明（docs/pr-description.md），明确本次新增或修改的条目数量、分类调整原因及构建输出是否产生预期变化。

5. 合并后 CI 流水线将自动构建并部署至预发布环境，维护者需在 48 小时内验证导航页链接可访问性，确认无误后手动触发生产部署。

## 常见问题

**问：为什么链接中同时存在 .sHtML 和 .html 混合大小写？**

答：所有 URL 均直接来源于上游数据源，我们不对原始文件扩展名做任何大小写归一化处理。部分 Web 服务器对大小写敏感，保留原始形式可确保跳转后正确访问目标资源。

**问：如何报告某个链接失效或内容与分类不符？**

答：请在 GitHub Issues 中提交问题，标题格式为 [Link Health] + 原始 URL 前 50 个字符。内容需附上 HTTP 状态码或重定向链信息，维护团队将在每批次复审周期内统一核查并标记。
