# LinkVault Resource Aggregator

LinkVault is a curated, high-density technical resource index system designed for developers, researchers, and content curators who need to organize, categorize, and rapidly access a large corpus of distributed web articles and reference materials. The project addresses the common challenge of managing hundreds of unstructured external links by providing a standardized metadata extraction pipeline, a lightweight indexing engine, and a static-site generation layer that transforms raw URL collections into a browsable knowledge base.

Target users include open-source maintainers building resource hubs, technical writers compiling reference lists, and engineering teams maintaining internal documentation portals. LinkVault does not host content directly but acts as a structured gateway, preserving original source integrity while adding searchability, tagging, and cross-reference capabilities on top of plain URL lists. The system is built to handle batches of up to 200 links per run, with the current release covering the 29th batch of a 56-batch ingestion pipeline, totaling over 180 active resource entries.

## 功能概览

**批量链接导入解析** - Supports recursive ingestion of plain-text URL lists, with automatic scheme normalization and duplicate detection across batches.

**元数据智能提取** - Fetches HTTP response headers and HTML title tags from each target URL to generate descriptive summaries without deep crawling.

**分类标签自动生成** - Applies rule-based categorization using domain patterns and path heuristics to assign each resource to predefined topic buckets.

**索引状态可视化面板** - Renders a dashboard showing ingestion progress, response status codes, and last-checked timestamps for every tracked link.

**静态站点一键导出** - Exports the entire indexed resource collection as a standalone HTML site with alphabetical and chronological sorting options.

**增量更新与变更追踪** - Detects changes in target response statuses and updates the index incrementally, avoiding full re-ingestion on each run.

**搜索过滤与全文检索** - Provides client-side search over titles, categorized tags, and source domain metadata using a lightweight inverted index.

**批量健康检查调度** - Runs scheduled HEAD requests against all stored URLs to validate liveness and flag broken links for manual review.

## 应用场景

**技术文档库维护** - A development team maintaining an internal wiki can use LinkVault to aggregate external references from blog posts, specification documents, and API changelogs. Each ingested URL is automatically enriched with its title and last-modified header, enabling the team to quickly verify whether external dependencies have been updated.

**学术资源整理** - Researchers collecting supplementary materials from multiple conference proceedings and preprint servers can import their bookmark lists into LinkVault. The system preserves the original article links while adding local tags such as "peer-reviewed", "dataset", or "supplement", making it easy to generate bibliography-style export pages for collaboration with peers.

**开源项目外链集中管理** - An open-source project with extensive documentation can replace scattered "References" sections with a single LinkVault-managed index. Contributors can submit new resource URLs via pull requests to a plain-text file, and the build pipeline regenerates the reference page automatically, ensuring all external links are checked for availability at build time.

**运维监控知识库** - Site reliability engineers can curate a private collection of troubleshooting guides, runbook templates, and incident post-mortems hosted across various internal and external platforms. LinkVault's scheduled health checks alert the team when a critical reference URL becomes unreachable, allowing proactive updates to operational documentation.

**个人开发者书签系统** - Individual developers can replace browser bookmark managers with a version-controlled repository of resource links. LinkVault generates a clean, searchable HTML page that can be hosted on GitHub Pages or any static hosting service, providing a portable and publicly shareable reference library.

## 快速开始

Clone the repository, install dependencies, and run the ingestion pipeline using the following commands in your terminal:

```bash
git clone https://github.com/example/linkvault.git
cd linkvault
pip install -r requirements.txt
python run.py --batch ./data/batch_29.txt --output ./dist/index.html
```

For custom configuration, copy `config.example.yaml` to `config.yaml` and adjust the output directory, request timeout, and concurrency limits. The batch file should contain one URL per line. After execution, open `./dist/index.html` in any modern browser to view the indexed resource list.

## 项目结构

```
linkvault/
├── run.py                      # 主入口脚本，解析命令行参数并启动流程
├── config.yaml                 # 用户配置文件，设置超时、并发数、输出路径
├── requirements.txt            # Python 依赖列表，用于 pip 安装
├── data/
│   ├── batch_29.txt            # 当前批次原始 URL 列表，每行一个链接
│   ├── batch_30.txt            # 下一批次预留文件
│   └── index.db                # SQLite 数据库，存储所有已处理链接的元数据
├── src/
│   ├── fetcher.py              # HTTP 请求模块，负责获取响应头与 HTML 标题
│   ├── parser.py               # HTML 解析模块，使用 BeautifulSoup 提取 title 和 meta
│   ├── indexer.py              # 索引核心逻辑，将元数据写入数据库并更新状态
│   ├── categorizer.py          # 分类规则引擎，基于域名和路径模式分配标签
│   ├── exporter.py             # 导出模块，渲染 Jinja2 模板生成静态 HTML
│   └── scheduler.py            # 健康检查调度，定期执行 HEAD 请求验证链接可用性
├── templates/
│   ├── base.html               # 基础 HTML 模板，包含样式与布局框架
│   └── index.html              # 资源列表页模板，渲染分类表格与搜索框
├── docs/
│   ├── quickstart.md           # 快速入门文档，含安装与首次运行步骤
│   ├── operations.md           # 运维手册，涵盖备份、恢复与性能调优
│   ├── development.md          # 开发者指南，讲解如何添加新的解析器后端
│   └── architecture.md         # 系统架构设计文档，含流程图与模块依赖图
├── tests/
│   ├── test_fetcher.py         # 单元测试，验证 HTTP 请求处理逻辑
│   ├── test_parser.py          # 单元测试，验证 HTML 标题提取准确性
│   └── test_indexer.py         # 单元测试，验证数据库读写操作
└── dist/
    └── index.html              # 最终生成的静态站点，可直接在浏览器中打开
```

## 贡献指南

**提交新的资源列表批次** - Fork 本仓库，在 `data/` 目录下创建新的批次文件（如 `batch_30.txt`），每行一个完整 URL，确保不包含空行或多余空格，然后提交 Pull Request。维护者会运行验证脚本检查 URL 格式与可访问性。

**改进核心解析逻辑** - 如果发现某个目标站点的 HTML 结构特殊导致标题提取失败，请修改 `src/parser.py` 并补充对应的测试用例到 `tests/test_parser.py`。提交前请运行 `pytest tests/` 确保所有现有测试通过。

**优化输出模板样式** - 对生成的静态页面有样式或布局改进建议时，请编辑 `templates/base.html` 和 `templates/index.html`。注意保持轻量化，避免引入外部 CDN 资源，确保页面完全离线可用。

**完善文档内容** - 文档位于 `docs/` 目录，接受拼写修正、示例补充和操作步骤澄清类的贡献。请使用 Markdown 语法，并保持与技术实现细节的一致性，避免过时描述。

**报告问题与提议功能** - 使用 GitHub Issues 提交 Bug 报告或功能请求。请包含环境信息（Python 版本、操作系统）、复现步骤和期望行为。对于功能提议，请说明使用场景和预期收益，以便核心维护者评估优先级。

## 常见问题

**问：如何处理某个链接无法访问或响应超时的情况？**

答：LinkVault 默认将超时设为 10 秒，连接错误或 4xx/5xx 状态码会被记录到数据库的 `last_status` 字段，并在导出的 HTML 中用颜色标记。您可以通过 `config.yaml` 中的 `timeout` 和 `retry_count` 参数调整超时和重试策略。对于永久失效的链接，建议从批次文件中移除或手动标记为忽略。

**问：是否支持 HTTPS 站点？我注意到部分链接使用 HTTP。**

答：LinkVault 完全支持 HTTPS 和 HTTP 两种协议，按原样使用 URL 中的 scheme。如果目标站点强制跳转到 HTTPS，`fetcher.py` 会自动跟随重定向，最终响应的 URL 会被记录到 `final_url` 字段。您无需手动修改协议前缀，系统会透明处理重定向链。

**问：如何将已有书签文件（如浏览器导出的 HTML）导入为批次文件？**

答：LinkVault 不直接解析浏览器书签格式，但您可以使用 `utils/bookmark_converter.py` 辅助脚本（位于 `contrib/` 目录）将 Netscape 格式的书签 HTML 转换为纯 URL 列表。转换后请检查输出文件，移除重复项，然后放入 `data/` 目录作为标准批次处理。
