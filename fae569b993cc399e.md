# WebArchive Navigator

WebArchive Navigator 是一个面向技术研究人员、内容归档工作者和数据分析师的开源外链资源汇总与导航工具。该项目系统性地收集并整理了来自 wap.blog.occzl.cn 的大量技术文章链接，覆盖软件开发、系统运维、前端工程、数据库管理、算法设计等多个技术领域，旨在为开发者提供一套结构清晰、可检索、可持续更新的技术阅读索引。

项目核心定位为"技术资源的外链汇总站"，不存储任何实际文章内容，仅提供链接索引与分类导航功能。目标用户包括需要快速查阅特定技术主题的研发工程师、希望系统化跟踪技术博客更新的内容消费者，以及从事技术文献计量分析的研究人员。通过本项目提供的结构化数据与辅助工具，用户可以显著降低在海量技术内容中发现高价值信息的时间成本。

## 功能概览

- **链接索引与分类检索**：按照技术领域、文章类型、发布时间等多维度对收录链接进行归类，支持用户通过命令行或 Web 界面进行快速筛选与检索。

- **批量导入与解析**：提供脚本工具，支持用户将自有链接列表批量导入系统，自动解析 URL 参数并提取元信息，生成标准化的索引记录。

- **去重与有效性校验**：内置链接去重算法和 HTTP 状态检查模块，可定期扫描索引库，标记失效链接并生成报告，确保资源列表的可用性。

- **自定义标签系统**：允许用户为每条链接添加自定义标签（如"必读""进阶""实战"等），构建个人化的知识分类体系，并支持标签云视图生成。

- **全文元数据搜索**：基于链接标题、来源域名、摘要关键词等元数据字段提供轻量级全文搜索能力，支持模糊匹配与布尔查询语法。

- **导出与集成支持**：支持将索引数据导出为 JSON、CSV、Markdown 表格等多种格式，便于嵌入其他文档系统、静态站点生成器或数据分析流水线。

- **增量更新机制**：通过版本化的索引文件管理，支持增量拉取最新收录链接，避免全量同步带来的网络与计算开销，适合定期自动化运维场景。

## 应用场景

- **技术团队内部知识库建设**：技术团队可将本项目作为知识库的底层链接引擎，将团队内部积累的优质外链统一导入系统，配合标签与搜索功能，构建团队共享的技术阅读清单，替代零散的浏览器书签。

- **技术博客的阅读队列管理**：日常需要跟踪大量技术博客的开发者，可以使用本项目维护个人阅读队列。通过定期运行有效性校验，及时发现失效链接；利用标签系统区分"待读""已读""精读"等状态，提升信息管理效率。

- **开源项目文档的外链引用整理**：开源项目维护者在编写文档时，常需引用大量外部参考资料。本项目提供的结构化导出功能可帮助维护者快速整理参考文献列表，并自动生成符合 Markdown 规范的链接表格，减少手动排版工作量。

- **技术文献的计量分析**：研究人员可利用本项目导出的结构化数据集，对特定域名下的文章发布频率、主题分布、链接生命周期等进行统计分析，用于技术趋势研判或学术研究中的实证数据支撑。

## 快速开始

以下步骤将帮助您在本地环境中快速启动 WebArchive Navigator 的核心服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/webarchive-navigator/webarchive-navigator.git
cd webarchive-navigator

# 安装项目依赖（使用 pip 进行 Python 依赖安装）
pip install -r requirements.txt

# 初始化索引数据库并导入首批链接数据
python scripts/init_db.py --source data/links_manifest.json
python scripts/import_links.py --input data/raw_links.txt --output data/links_manifest.json

# 启动本地导航服务（默认监听 127.0.0.1:8080）
python app.py --host 127.0.0.1 --port 8080
```

完成上述步骤后，在浏览器中访问 `http://127.0.0.1:8080` 即可开始使用本地导航界面。

```

├── app.py                         # Web 服务主入口，启动导航界面与 API 端点
├── requirements.txt               # Python 依赖声明文件，包含所有必需第三方库
├── config/
│   ├── settings.py                # 全局配置参数（端口、数据库路径、缓存策略等）
│   └── logging.conf               # 日志格式与输出级别配置
├── data/
│   ├── links_manifest.json        # 主索引清单文件，存储所有链接元数据（JSON 格式）
│   ├── raw_links.txt              # 原始链接导入模板，供批量添加使用
│   └── tags_index.db              # SQLite 标签索引数据库文件
├── scripts/
│   ├── init_db.py                 # 初始化数据库表结构与默认配置
│   ├── import_links.py            # 从 raw_links.txt 导入链接至 manifest
│   ├── validate_links.py          # 批量执行 HTTP 状态检查，更新链接可用性标记
│   └── export_formats.py          # 导出索引数据为 JSON / CSV / Markdown 等格式
├── core/
│   ├── parser.py                  # URL 解析与元信息提取核心模块
│   ├── dedup.py                   # 基于哈希的链接去重算法实现
│   ├── search.py                  # 轻量级全文搜索引擎（基于倒排索引）
│   └── tagger.py                  # 标签系统的增删改查与聚合统计逻辑
├── web/
│   ├── static/
│   │   ├── style.css              # Web 界面样式表
│   │   └── script.js              # 前端交互逻辑（搜索、标签筛选、分页）
│   └── templates/
```

## 常见问题

**问：本项目是否存储文章的实际内容？是否存在版权风险？**

答：本项目仅存储链接地址及其元数据（如标题、来源域名等），不缓存、不镜像、不存储任何原始文章内容。所有链接均指向原始第三方网站，用户访问内容时实际请求的是源服务器。本项目作为纯粹的导航索引工具，符合合理使用原则。如任何内容所有者认为链接索引侵犯其权益，可联系项目维护者申请从索引中移除。

**问：如何更新已收录链接的有效性状态？**

答：项目提供了 `scripts/validate_links.py` 脚本，可批量检查所有收录链接的 HTTP 状态码。建议用户定期运行该脚本（例如每周一次），脚本会自动更新 manifest 文件中每条链接的 `status` 字段和 `last_checked` 时间戳。运行命令为 `python scripts/validate_links.py --manifest data/links_manifest.json --timeout 10`，其中 `--timeout` 参数可调整单次请求的超时时间。

**问：我能否将本项目用于商业产品中？**

答：可以。本项目采用 MIT 许可证，允许自由使用、修改、分发和再授权，包括用于商业目的。唯一的条件是保留原始版权声明和许可声明。详细条款请参阅项目根目录下的 LICENSE 文件。
