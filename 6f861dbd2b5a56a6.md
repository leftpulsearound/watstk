# WebResource Indexer

WebResource Indexer 是一个面向技术研究人员、开发者与信息分析人员的结构化外链资源汇总系统。该项目不对原始内容进行二次加工，仅提供一致的访问入口与分类导航，帮助用户从大量分散的移动端文章链接中快速定位潜在价值信息。

项目定位为轻量级资源索引工具，核心目标用户包括：需要批量处理移动端文章链接的数据采集工程师、进行内容聚合分析的产品运营人员、以及希望系统化整理个人学习资料库的开发者。通过统一的索引格式与清晰的分类逻辑，WebResource Indexer 将原本离散的 URL 转化为可维护、可共享、可扩展的知识索引集合。

## 功能概览

**批量链接收录**：支持一次性导入数百条移动端文章链接，保留原始 URL 完全不变，确保访问路径的绝对准确性。

**自动分类归档**：根据 URL 中的数字 ID 特征与路径结构，对链接进行自动分组，生成初步的类别标签与归档目录。

**索引状态标记**：为每条链接记录收录批次、收录时间与校验状态，方便后续跟踪链接可用性变化。

**快速检索支持**：基于链接 ID 与收录批次号提供简单的文本匹配检索，帮助用户在大量链接中快速筛选目标条目。

**原始数据导出**：支持将索引数据导出为纯文本列表、CSV 表格或 JSON 结构化文件，便于对接其他数据处理管道。

**可扩展架构**：提供清晰的配置文件与插件接口，允许用户自定义分类规则、添加额外元数据字段或接入外部 API 进行内容解析。

## 应用场景

**移动端技术文章批量整理**：研究人员在浏览移动端博客平台时，可批量收集感兴趣的文章链接，通过 WebResource Indexer 建立临时索引，避免浏览器书签杂乱无章，同时保留完整的原始 URL 用于后续深度阅读。

**数据采集管道前置处理**：数据工程师在启动大规模爬虫任务前，使用本项目对目标链接列表进行去重、分类与状态预检，生成结构化的任务清单，提高采集任务的执行效率与可维护性。

**内容聚合平台素材管理**：内容运营人员从多个来源获取移动端文章链接后，利用本项目的归档功能将链接按主题或批次分组，为后续的内容二次编辑、翻译或评论提供有序的素材库。

**个人知识库外部引用整理**：开发者在撰写技术博客或整理学习笔记时，将引用的外部移动端文章链接通过本项目统一管理，确保引文来源清晰可查，减少链接失效带来的信息丢失风险。

## 快速开始

以下步骤将帮助您在本地环境快速启动 WebResource Indexer 实例。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/webresource-indexer.git

# 进入项目目录
cd webresource-indexer

# 安装依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 初始化索引数据库
python manage.py initdb

# 导入示例链接列表（使用项目自带的 sample_links.txt）
python manage.py import --file sample_links.txt --batch 31

# 启动本地 Web 服务（默认端口 8080）
python manage.py runserver --port 8080
```

访问 http://localhost:8080 即可查看当前批次的链接索引看板。

## 项目结构

```
webresource-indexer/
├── manage.py                 # 项目统一管理入口，集成导入、导出、服务启动等命令
├── requirements.txt          # Python 依赖清单，包含 Flask、SQLAlchemy 等核心库
├── config/
│   ├── default.py            # 默认配置项，包含数据库路径、服务端口、批次号等
│   └── custom.py             # 用户自定义配置模板，用于覆盖默认行为
├── core/
│   ├── __init__.py           # 核心模块初始化
│   ├── indexer.py            # 链接索引核心逻辑，包含导入、去重、分类功能
│   ├── validator.py          # 链接校验模块，负责检查 URL 格式与可达性
│   └── exporter.py           # 数据导出模块，支持 TXT、CSV、JSON 格式输出
├── web/
│   ├── __init__.py           # Web 模块初始化
│   ├── app.py                # Flask 应用主程序，定义路由与视图函数
│   ├── templates/            # HTML 模板目录，存放索引看板与详情页面
│   │   ├── base.html         # 基础模板，包含公共样式与导航
│   │   ├── index.html        # 首页模板，展示批次汇总与链接列表
│   │   └── detail.html       # 单条链接详情模板，展示完整元数据
│   └── static/               # 静态资源目录，包含 CSS 样式与 JavaScript 脚本
├── data/
│   ├── index.db              # SQLite 数据库文件，存储所有链接索引数据
│   └── batches/              # 批次导入原始文件存档目录
│       └── batch_31.txt      # 第 31 批次原始链接列表存档
├── docs/                     # 项目文档目录，包含用户指南与开发者手册
│   ├── user-guide.md         # 用户使用指南
│   ├── admin-manual.md       # 管理员运维手册
│   └── developer-guide.md    # 开发者扩展文档
├── tests/                    # 单元测试目录
```

## 常见问题

问：导入链接时提示校验失败，但链接在浏览器中可以正常访问，是什么原因？

答：默认的链接校验模块会发送 HTTP HEAD 请求检查响应状态码。部分移动端站点可能不支持 HEAD 方法或返回非标准状态码，导致校验误判。您可以在 config/custom.py 中将 VALIDATE_ON_IMPORT 设置为 False 以跳过校验，或在 validator.py 中调整允许的状态码列表。

问：如何将本项目部署到生产环境供团队内部使用？

答：生产环境部署建议使用 Gunicorn 或 uWSGI 作为 WSGI 服务器，并在前端配置 Nginx 反向代理。具体部署步骤请参考 docs/admin-manual.md 中的生产环境部署章节，其中包含 systemd 服务配置模板与日志轮转方案。

问：索引数据库中的数据如何备份与迁移？

答：SQLite 数据库文件位于 data/index.db，您可以直接复制该文件进行备份。迁移时只需将数据库文件复制到目标环境的 data/ 目录下，并确保 config/default.py 中的数据库路径配置正确。若需要迁移到其他数据库系统（如 PostgreSQL），请参考开发者文档中的数据库适配器扩展指南。
