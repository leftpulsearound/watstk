# WapBlog Article Aggregator

WapBlog Article Aggregator 是一个面向技术内容聚合与归档的开源项目，专注于采集、整理和检索来自移动端博客平台的历史技术文章。该项目主要服务于技术研究者、内容归档工程师以及博客平台迁移运维人员，帮助其系统性地获取 wap.blog.occzl.cn 平台上散落的深度技术文章，并建立本地化的检索与阅读环境。

该项目并非一个传统的博客前端或 CMS 系统，而是一个以数据管道为核心的工具集，包含文章抓取调度、内容解析清洗、元数据提取与结构化存储等模块。通过本项目，用户可以将分散在数百个动态 URL 下的技术文档统一纳入本地知识库，支持全文检索、标签分类与离线阅读，有效解决移动端博客内容难以批量管理、链接失效后无法追溯的问题。

## 功能概览

**批量文章抓取**：基于提供的文章 ID 列表，自动生成完整请求路径，支持多线程并发下载，显著缩短大规模归档任务的总耗时。

**内容结构化解析**：针对 wap.blog.occzl.cn 的 HTML 结构进行适配，自动提取文章标题、正文、发布时间、分类标签及作者信息，输出为统一的 JSON 或 Markdown 格式。

**断点续传与任务恢复**：抓取任务支持中断恢复机制，程序自动记录已完成的文章 ID，重启后跳过已处理条目，避免重复劳动。

**增量更新策略**：支持定时轮询机制，可配置检测新发布文章并仅拉取增量内容，适合长期维护型归档需求。

**本地全文检索引擎**：内置轻量级倒排索引，支持中文分词与布尔查询，不依赖外部搜索引擎即可实现毫秒级检索响应。

**导出与迁移工具**：提供将归档数据导出为 SQLite 数据库、静态 HTML 站点或 WordPress XML 导入格式的转换脚本，便于与其他平台对接。

## 应用场景

**个人技术博客的本地备份**：当开发者需要将 wap.blog.occzl.cn 上自己或感兴趣作者的历史文章保存到本地硬盘时，本项目提供全自动化的抓取与整理流程，避免因平台关闭或链接失效导致的内容损失。

**企业知识库的数据迁移**：技术团队在迁移内部 Wiki 或文档中心时，可使用本项目将外部参考文章批量拉取并转换为 Markdown 格式，统一纳入 Git 仓库进行版本管理。

**学术研究的数据采集**：研究人员需要分析移动端技术博客的写作风格、热门主题演变或链接引用网络时，本项目提供稳定的数据采集管道，可输出结构化数据集供后续 NLP 分析。

**网站内容监控与死链检测**：运维人员可配置本项目定期抓取指定文章列表，通过比对 HTTP 状态码和内容哈希值，及时发现文章删除、篡改或链接失效情况。

## 快速开始

以下命令演示了如何在 Linux 或 macOS 环境下完成项目的克隆、依赖安装与首次运行。

```bash
# 克隆项目仓库到本地
git clone https://github.com/your-org/wapblog-aggregator.git
cd wapblog-aggregator

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 运行示例抓取任务（默认使用提供的文章 ID 列表）
python main.py --input urls.txt --output ./data --threads 4
```

其中 `urls.txt` 为每行一个文章 ID 的文本文件，程序会自动拼接为完整的 wap.blog.occzl.cn 链接进行请求。

## 常见问题

问：抓取过程中遇到 HTTP 403 或 429 状态码如何解决？

答：此类状态码通常表示服务端实施了访问频率限制或 User-Agent 校验。建议首先降低抓取线程数（通过 --threads 参数调至 2 或 1），并在 config.yaml 中增大请求间隔时间（delay_seconds 配置项）。同时确认 User-Agent 设置为主流浏览器标识，避免被识别为自动化脚本。若问题持续，可尝试更换出口 IP 或使用代理池。

问：如何导入自定义的文章 ID 列表进行批量归档？

答：项目支持两种导入方式。其一是直接在命令行通过 --ids 参数传递逗号分隔的 ID 列表，例如 --ids 56227,76521,8056。其二是准备一个纯文本文件，每行一个 ID，然后使用 --input 参数指定文件路径，如 --input my_ids.txt。程序会自动读取并去重，同时跳过已归档的 ID。

问：解析后的文章正文出现乱码或内容缺失怎么办？

答：该问题通常源于目标页面的 HTML 结构变动或编码声明不一致。首先检查 config.yaml 中的 encoding 配置，尝试切换为 gbk 或 utf-8。若仍无法解决，请运行 scripts/debug_parser.py 脚本输出原始 HTML 样本，对比 parser/html_cleaner.py 中的选择器是否匹配当前页面结构。如确认是平台改版导致，欢迎提交包含样本 URL 的 Issue，维护者会尽快适配新的解析规则。
